## 3. 전체 데이터 흐름

```
수신: /charger/prices
    │
    ├─ [Step 1] DB PriceSchedules 테이블에 Upsert
    │           station_id, charger_id(저장만), create_date, apply_date, end_date, H00~H23
    │           null 값 자동 처리:
    │             applyDate  null → DateTime.Now
    │             createDate null → DateTime.Now
    │             endDate    null → "99991231235959" (무기한)
    │
    ├─ [Step 2] 즉시 활성 스케줄 판단
    │           조건: apply_date <= now AND end_date >= now (양쪽 모두 체크)
    │           우선순위: create_date DESC
    │           → 활성이면 INI(H00~H23) 갱신 + AppSettingsManager.Save()
    │           → DB KeyValues 자동 백업
    │
    ├─ [Step 3] 충전 중 채널 즉시 반영
    │           → 단가 변경 시 구간 이력(UnitCostChangeHistory) 저장
    │           → CurrentUserUnitCost 갱신
    │           → UI PowerMeter 재계산 트리거
    │
    └─ [Step 4] PriceScheduleTimer (매 1분 :00초 정각)
                시작 시:
                  → 다음 :00초까지 남은 시간으로 첫 Interval 설정
                  → 500ms 후 즉시 1회 실행 (시작 시 DB 스케줄 즉시 반영)
                  → AppSettingsManager.ApplyActivePriceSchedule(stationId) 호출 ✅
                매 Tick:
                  → HasAny(stationId) 확인 (레코드 없으면 공유 배열 수정 안 함)
                  → PurgeExpired(): 만료 후 1개월 경과 데이터만 삭제
                  → GetActive(): station_id 기준 활성 스케줄 조회
                  → 활성 스케줄 없고 기존 레코드 있었으면 → PriceForHour 347.2f 복원
                  → 단가 변경 시 INI 저장, 충전 중 채널 단가 전환
                  → AppSettingsManager.ApplyActivePriceSchedule(stationId) 호출 ✅
```

## 5. DB 스키마 변경

### 5.1 신규 테이블: `PriceSchedules`

```sql
CREATE TABLE IF NOT EXISTS PriceSchedules (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    station_id  TEXT NOT NULL,
    charger_id  TEXT NOT NULL,   -- 수신 데이터 그대로 저장 (조회는 station_id 기준)
    create_date TEXT NOT NULL,   -- 우선순위 기준 (YYYYMMDDHHmmss)
    apply_date  TEXT NOT NULL,   -- 단가 적용 시작일시
    end_date    TEXT NOT NULL,   -- 단가 적용 종료일시
    prices_json TEXT NOT NULL,   -- H00~H23 JSON 배열
    created_at  TEXT NOT NULL DEFAULT (datetime('now'))
);

-- UNIQUE INDEX: station_id + create_date 조합으로 중복 방지 (charger_id는 저장만, 조회/중복검사에 미사용)
CREATE UNIQUE INDEX IF NOT EXISTS UX_PriceSchedules
    ON PriceSchedules(station_id, create_date);
CREATE INDEX IF NOT EXISTS IX_PriceSchedules_Apply
    ON PriceSchedules(station_id, apply_date, end_date);
```

### 5.2 기존 테이블 유지

| 테이블 | 역할 |
|--------|------|
| `KeyValues` | settings.ini 전체 백업/복원 |
| `TransmissionLog` | 전문 송신 이력 (오프라인 재전송) |
| `ChargingSessions` | 충전 세션 이력 |
| `PriceSchedules` | **단가 스케줄 저장 (신규)** |

## 6. 상세 구현

### 6.1 `PriceScheduleRepository.cs` (신규, station_id 기준)

```
핵심 메서드 (모든 조회/삭제는 station_id 기준, charger_id 조건 없음)

Upsert(stationId, chargerId, createDate, applyDate, endDate, prices[])
  → charger_id는 수신 데이터 그대로 저장 (조회에는 사용 안 함)
  → 동일 station_id/charger_id/create_date 이면 INSERT OR REPLACE (idempotent)
  → applyDate/endDate/createDate null·빈문자열 자동 처리:
      applyDate  null → DateTime.Now
      createDate null → DateTime.Now
      endDate    null → "99991231235959" (무기한)

double[] GetActive(stationId, nowStr)
  → apply_date <= now <= end_date 조건 중 create_date DESC LIMIT 1
  → station_id 기준 (charger_id 구분 없음)

string GetNextApplyDate(stationId, nowStr)
  → 타이머 전환 감지용: 가장 가까운 미래 apply_date 반환
  → station_id 기준

void PurgeExpired(stationId, nowStr)
  → end_date < (현재 - 1개월) 인 만료 스케줄만 삭제 (즉시 삭제 안 함)
  → station_id 기준

bool HasAny(stationId)
  → 해당 충전소에 스케줄 레코드가 하나라도 존재하는지 확인 (만료 포함)
  → station_id 기준
  → PurgeExpired 호출 전에 사용 (공유 배열 보호 판단용)
```

### 6.2 `AppSettingsManager.ApplyActivePriceSchedule(stationId)`

```
동작 순서
0. HasAny(stationId) 호출 → 레코드 존재 여부 미리 기록
1. PurgeExpired(stationId, nowStr) 호출 → 1개월 경과 만료 스케줄 정리
2. GetActive(stationId, nowStr) 호출
3. activePrices == null 이면:
   - HasAny == false  → DB에 해당 충전소 레코드 자체 없음
                       → 공유 PriceForHour 배열 건드리지 않고 return false
   - HasAny == true   → 스케줄 있었으나 현재 활성 없음 (만료)
                       → PriceForHour 전체를 기본값 347.2f 로 복원 후 Save()
4. activePrices 있으면:
   - PriceForHour[i] vs activePrices[i] 비교
   - 변경된 경우에만 Save() 호출
     → settings.ini H00~H23 갱신
     → DB KeyValues 자동 백업
5. 변경 여부(bool) 반환
```

### 6.3 `CallBackResponseDataPrices()` 처리 순서

```
① create_date, apply_date, end_date 파싱
② PriceScheduleRepository.Upsert()   → DB 저장
③ 즉시 적용 조건 판단:
   apply_date <= now AND end_date >= now 일 때만 즉시 반영
④ ChangeChargingUnitPrices() 호출    → INI 갱신 + 로그
⑤ 충전 중 채널 즉시 반영 (구간 이력 저장 + CurrentUserUnitCost 갱신 + UI 갱신)
⑥ response = "1" 반환
```

### 6.4 `PriceScheduleTimer` (매 1분 정각 :00초)

```
초기화:
  - 다음 :00초까지 남은 시간 = (60 - now.Second) * 1000 - now.Millisecond 으로 첫 Interval 설정
  - 이후 매 Tick마다 남은 시간 재계산 → Interval 재조정 (누적 오차 제거)

프로그램 시작 시:
  - 500ms 지연 후 즉시 ApplyActivePriceSchedule() 1회 실행
    (타이머 첫 Tick은 다음 정각이므로 시작 시 즉시 DB 스케줄 반영)

매 Tick (:00초마다):
  stationId = _channels[0].StationId  (station_id 기준, 채널 구분 없음)
  ApplyActivePriceSchedule(stationId) 호출
  → 단가 변경 시 INI 저장
  → 충전 중이면 CurrentUserUnitCost 갱신
```

## 7. 시간대별 할인 요금 정산 방법

### 7.1 할인 적용 정책 (봄·가을철 토요일·공휴일 11시~14시)

| 구분 | 100kW 미만 | 100kW 이상 |
|------|-----------|-----------|
| 기존 충전요금 | 324.4원 | 347.2원 |
| 할인 충전요금 (토요일) | 48.6원 (15.0%) | 48.6원 (14.0%) |
| 할인 충전요금 (공휴일) | 42.7원 (13.2%) | 42.7원 (12.3%) |
| **할인적용 충전요금 (토요일)** | **275.8원** | **298.6원** |
| **할인적용 충전요금 (공휴일)** | **281.7원** | **304.5원** |

> ※ 임시 공휴일 제외

### 7.2 충전 시간에 따른 구간별 계산 방법

| 사례 | 충전시간 | 계산 방법 |
|------|---------|----------|
| 예1 | 10시~10시58분 (할인 미적용) | 10시~10시58분 : 충전량 × 할인전 단가 |
| 예2 | 10시~13시 (할인 적용) | 10시~10시59분59초 : 충전량 × 할인전 단가<br>**11시00분00초~13시 : 충전량 × 할인 단가** |
| 예3 | 13시58분~14시10분 (할인 적용) | 13시58분01초~14시00분00초 : 충전량 × 할인전 단가<br>**14시00분01초~14시10분 : 충전량 × 할인 단가** |

### 7.3 구간 계산 로직 (`UnitCostChangeHistory`)

```
충전 시작 시:
  CurrentSegmentStartPowerMeter       = 현재 PowerMeter
  AccumulatedCostBeforeCurrentSegment = 0
  UnitCostChangeHistory               = 초기화

단가 변경 이벤트 발생 시 (CallBackResponseDataPrices or PriceScheduleTimer):
  segmentCost = (currentPowerMeter - CurrentSegmentStartPowerMeter) × oldUnitCost
  totalCost   = AccumulatedCostBeforeCurrentSegment + segmentCost

  UnitCostChangeHistory.Add({
    PowerMeter      = currentPowerMeter,
    UnitCost        = oldUnitCost,
    AccumulatedCost = totalCost
  })

  CurrentSegmentStartPowerMeter       = currentPowerMeter
  AccumulatedCostBeforeCurrentSegment = totalCost
  CurrentUserUnitCost                 = newUnitCost

충전 종료 시:
  최종 구간 = (currentPowerMeter - CurrentSegmentStartPowerMeter) × CurrentUserUnitCost
  최종 총액 = AccumulatedCostBeforeCurrentSegment + 최종 구간
  원단위 절삭 (10원 단위 내림) 처리
```

## 9. 주의사항 및 제약

| 항목 | 내용 |
|------|------|
| **SQLite 버전** | `ON CONFLICT` 문법 미지원 환경 대비 → `INSERT OR REPLACE` 방식 사용 |
| **.NET Framework 4.6** | `async/await`, `LINQ`, `nameof` 사용 가능 |
| **타이머 스레드** | `PriceScheduleTimer` 는 Dispatcher(UI 스레드)에서 실행 → DB 접근 시 블로킹 주의 |
| **오프라인 대응** | DB에 스케줄 저장되어 있으므로 네트워크 단절 시에도 타이머로 자동 적용 가능 |
| **apply_date 없는 수신** | `apply_date` 가 공란이면 즉시 적용으로 처리 (`now` 기준) |
| **end_date 없는 수신** | `end_date` 가 공란이면 `99991231235959` (무기한)으로 처리 |
| **만료 스케줄 정리** | `PurgeExpired()` — 만료 후 **1개월 경과** 데이터만 삭제 (즉시 삭제 안 함) |
| **기본 단가** | 스케줄 없거나 만료 시 `PriceForHour` 전체를 **347.2f** 로 자동 복원 |
| **단가 기준 단위** | station_id 기준으로 처리 (charger_id 조건 없음, 채널 구분 없이 동일 단가 적용) |
| **공유 배열 보호** | DB에 해당 충전소 레코드 자체가 없으면 `PriceForHour` 배열 수정 안 함 |
| **타이머 정각 맞춤** | 시작 시 다음 :00초까지 남은 시간으로 첫 Interval 설정, 매 Tick마다 재조정 |
